Why Nash Isn't Enough

The Nash Analysis page proves what game theory already knows: in a one-shot Prisoner's Dilemma, mutual defection is the unique Nash equilibrium. Defect is strictly dominant for both players. The payoff at equilibrium is (0, 0) — while mutual cooperation yields (2, 2).

This is the Price of Anarchy in miniature. The Nash equilibrium exists, but it's a terrible outcome. Knowing where the fixed point is tells us nothing about how to reach a better one, or whether the system will converge to anything at all.

From Fixed Points to Dynamics

Nash equilibria are static — they describe where no one wants to deviate, not how a system gets there. The moment we introduce repeated play, population dynamics, and evolutionary selection (as the tournament does), the question shifts from "what are the equilibria?" to "what does the system actually do over time?"

The answer depends on parameters: how much noise corrupts signals, how long the shadow of the future extends. The same set of strategies can converge to cooperation or collapse into defection depending on these conditions. Nash analysis can't see this — it operates on the static 1v1 game, blind to the dynamics the tournament introduces.

The Simulation–Optimization Pipeline

The Simulation page shows individual state trajectories — what happens for one specific simulation configuration (a fixed noise and rounds value). But each configuration is just one point in a vast parameter space. How do you know which point to pick?

PSUU answers this by building the config → KPI mapping. Each evaluation runs a Monte Carlo batch — multiple stochastic simulations of the same configuration with different random seeds. Metrics (cooperation rate, diversity, winner share) are computed per trajectory, then aggregated across runs to produce KPIs that summarize the configuration's performance under uncertainty.

The scatter plot below visualizes this mapping: each point is one evaluation — a configuration and its resulting KPIs. The surface this creates is noisy, non-linear, and expensive to compute. PSUU explores it using random search, surfacing the configurations that optimize each KPI.

Parameter Search as Descent

This connects to the deeper argument in From Nash to Lyapunov: if we can define a scalar objective function (a KPI like cooperation rate), then searching the parameter space is a form of descent over a potential landscape. Each evaluation is a sample of that surface. More sophisticated optimizers — Bayesian optimization, evolutionary algorithms — could exploit its structure, but even random search reveals the landscape's shape.

This is the practical bridge between game theory and dynamical systems. Nash tells us defection is rational. PSUU asks: under what conditions does the evolutionary system overcome that rationality trap?
